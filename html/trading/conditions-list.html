  • 个别股票

    • 个股点差计算

      除非另有说明，个股交易条款显示个股交易品种“与市场的差价”。所谓“与市场的差价”，是AVATRADE在当前市场点差的基础上增加的幅度。

      点差成本公式：差价x交易头寸=收取的、以货币品种计价的点差费用
      例1

      0.12 X 1 = $0.12*

      交易1股苹果，差价为12个点子(0.12)，计算如下：

      0.12 X 1 = 0.12美金*

      *0.12美金是美元数量，因为点子以个股计价的货币品种计算。如果投资者的账户以非美元货币计价，系统将自动将点差费用按当前市场汇率转换成账户货币。

      例2

      交易10股安联，差价为150个点子(0.150) ，计算如下：

      0.150 X 10 = 1.50欧元*

      *1.50欧元是欧元数量，因为点子以个股计价的货币品种计算。如果投资者的账户以非欧元货币计价，系统将自动将点差费用按当前市场汇率转换成账户货币。

      例3

      交易100股汇丰，差价为80个点子(0.80) ，计算如下：

      0.80 X 100 = 80.0 或0.80英镑*

      （英国股票以便士计价，所以要除以100，即80/100)

      *0.80英镑是英镑数量，因为点子以个股计价的货币品种计算。如果投资者的账户以非欧元货币计价，系统将自动将点差费用按当前市场汇率转换成账户货币。

      所有个股的差价和计价的货币品种都可以在AVATRADE交易条款中找到，请看上述表格。

                                 AVATRADE是做市商，除非另有说明，否则透过买卖点差获利。AVATRADE对任何交易都不收取佣金。

    • 个股保证金计算

      所有保证金交易的品种使得投资者可以放大交易头寸。

      个股交易条款以百分比(%)显示保证金数字。

      百分比保证金公式：交易头寸x市价x保证金(%)=保证金要求*

      *保证金以计价货币的品种计算。

      AVA可能在利润报告公布之前将某些个股的保证金要求翻倍。这是预防性措施，以避免该个股的大客户头寸净值变为负数。

      例1

      交易1股苹果，市价为500美金，保证金要求为5.00%，计算如下：

      百分比保证金要求： 1 x 500 x 0.05 = 25.00美金*

      *25美金是美元数量，因为保证金以计价的货币品种计算。如果投资者的账户以非美元货币计价，系统将自动将保证金要求按当前市场汇率转换成账户货币。

      例2

      交易10股安联，市价为102.50欧元，保证金要求为10.00%，计算如下：

      百分比保证金要求：10 x 102.50 x 0.10 = 102.50欧元*

      *102.50欧元是欧元数量，因为保证金以计价的货币品种计算。如果投资者的账户以非欧元货币计价，系统将自动将保证金要求按当前市场汇率转换成账户货币。

      例3

      交易100股汇丰，市价为650.50便士，保证金要求为10.00，计算如下：

      百分比保证金要求：100 x 650.50 x 0.10 = 6,505.00 便士 或 65.05英镑*

      （英国股票以便士计价，所以要除以100，即6505/100)

      65.05英镑是英镑数量，因为保证金以计价的货币品种计算。如果投资者的账户以非英镑货币计价，系统将自动将保证金要求按当前市场汇率转换成账户货币。

      所有个股交易品种的保证金要求都可以在AVATRADE交易条款中找到，请看上述表格。

    • 个股隔夜利息计算

      个股交易条款显示，以360天为基础，在我们的日结算日后继续持有未平仓头寸，需要支付/可以得到的隔夜利息。这些利率显示在“溢价买”和“溢价卖”栏。日结算日指格林威治标准时间22:00，夏令时则调整为格林威治标准时间21:00。

      用公布的溢价，按以下公式计算自己的每日溢价数额：

      数量x当前价格x溢价买或卖率x交易天数=支付/收取的溢价*360天

      *支付/收取的溢价计价的货币品种计算。

      例1

      交易1股苹果，市场价格是500美金，溢价买（或卖）率是-2.55%，并且是1天的金额，计算如下：

      (1 x 500 x -0.0255 x 1)/360 = -12.75/360 = -0.03542 = -0.04美金*（四舍五入）

      *-0.04美金是美元数量，因为溢价以计价的货币品种计算。如果投资者的账户以非美元计价，系统将自动将隔夜费用按当时市场汇率转换成账户货币。

      例2

      交易10股安联，市场价格是102.50欧元，溢价买（或卖）率是-3.45%，并且是1天的金额，计算如下：

      (10 x 102.50 x -0.0345 x 1)/360 = -35.363/360 = -0.09823 = -0.10欧元*（四舍五入）

      *-0.10欧元是欧元数量，因为溢价以计价的货币品种计算。如果投资者的账户以非欧元计价，系统将自动将隔夜费用按当时市场汇率转换成账户货币。

      例3

      交易100股汇丰，市场价格是650.50便士，溢价买（或卖）率是-1.85%，并且是1天的金额，计算如下：

      (100 x 650.50 x -0.0185x 1)/360 = -1,203.43/360 = -3.3428/100 = -0.03英镑*（四舍五入）

      （英国股票以便士计价，所以要除以100，即-3.3428/100)

      *-0.03英镑是英镑数量，因为溢价以计价的货币品种计算。如果投资者的账户以非英镑计价，系统将自动将隔夜费用按当时市场汇率转换成账户货币。

      所有个股的买/卖溢价率及计价货币品种都可以在AVATRADE交易条款中找到，请看上述表格。

      AVATRADE在计算买/卖溢价时，标准做法是在隔夜市场贷款利率的卖价上减去30个基点以及在买价上增加30个基点；这些利率定期更新。请注意某些溢价计算包含较高的加价。

    • 个股法人行为与计算

      个股在某个阶段可能参与到公司行为；这些公司行为可以包括分红、权利股发行、股票合并/分拆、合并、兼并和收购等。

      分红：AVATRADE交易平台上的任何个股宣布分红时，AVATRADE将在附有红利日结束时，即除息日前一天，对持有该股的每个账户进行调整。

      上述调整将是：

      1.多头头寸将得到总分红的90%

      （持有的股票数量x总分红）x0.90

      2.空头头寸将扣除总分红的100%

      （持有的股票数量x总分红）x-1

      请注意：涉及分红时，客户没有其它费用。

      例1

      交易1股票苹果，总分红为1美金，计算如下：

      多头头寸：(1 x 1.00) x 0.90 = 1.00 x 0.90 = +0.90美金

      空头头寸：(1 x 1.00) x -1 = 1.00 x -1 = -1.00美金

      所有分红调整以计价的货币工具计算。如果阁下的账户与计价的货币工具不同，系统将自动按照当时的市场汇率将分红转化为账户货币。

      例2

      交易10股安联，总分红0.14欧元，计算如下：

      多头头寸：(10 x 0.14) x 0.90 = 1.40 x 0.90 = +1.26欧元

      空头头寸：(10 x 0.14) x -1 = 1.40 x -1 = -1.40欧元

      所有分红调整以计价的货币工具计算。如果阁下的账户与计价的货币工具不同，系统将自动按照当时的市场汇率将分红转化为账户货币。

      例3

      交易100股汇丰，总分红0.04英镑，计算如下：

      多头头寸：(100 x 0.04) x 0.90 = 4.00 x 0.90 = +3.60英镑

      空头头寸：(100 x 0.04) x -1 = 4.00 x -1 = -4.00英镑

      所有分红调整以计价的货币工具计算。如果阁下的账户与计价的货币工具不同，系统将自动按照当时的市场汇率将分红转化为账户货币。

  • 外汇标准

    • 外汇标准点差计算

      除非另有说明，外汇固定交易条款显示外汇交易品种的标准买卖价差（点子）。标准点差按照正常市况的规定。根据市况的不同，点差可以扩大，最高可到标准点差的三倍。

      点差成本公式：点差x交易头寸=收取的计价货币*点差费用

      *计价货币是一个外汇货币对中的第二个报价货币（第一个货币/第二个货币：美元/日元中的日元，欧元/美元中的美元，等等）。

      例1

      在1,000欧元/美元交易中，点差为3个点子(0.0003)，则计算如下：

      0.0003 X 1,000 = 0.30美金*

      *0.30美金是美元数量，因为点子以计价货币计算。在这个例子中，美元是欧元/美元货币对中的计价货币（欧元=基础货币，美元=计价货币）。如果投资者的账户以非美元货币计价，系统将自动将点差费用按当前市场汇率转换成账户货币。

      例2

      1,000美元/日元交易中，点差为4个点子(0.04)，计算如下：

      0.04 X 1,000 = 40.00日元*

      *40.00日元是日元数量，因为点子以计价货币计算。在这个例子中，日元是美元/日元货币对中的计价货币（美元=基础货币，日元=计价货币）。如果投资者的账户以非日元货币计价，系统将自动将点差费用按当前市场汇率转换成账户货币。

      例3

      0.0012 X 1,000 = C$1.20*

      1,000英镑/加元交易中，点差为12个点子(0.0012)，计算如下：

      (0.0012) X 1,000 = 1.2加元*

      *1.20加元是加元数量，因为点子以计价货币计算。在这个例子中，加元是英镑/加元货币对中的计价货币（英镑=基础货币，加元=计价货币）。如果投资者的账户以非加元货币计价，系统将自动将点差费用按当前市场汇率转换成账户货币。

      所有外汇固定交易品种的点差都可以在AVATRADE交易条款中找到，请看上述表格。

                                 AVATRADE是做市商，除非另有说明，否则透过买卖点差获利。AVATRADE对任何交易都不收取佣金。

    • 外汇标准保证金杠杆交易

      所有保证金交易的品种使得投资者可以放大交易头寸。外汇固定交易条款显示保证金和杠杆数字；保证金以百分比呈现，而杠杆以某个比值呈现。

      百分比保证金公式：交易头寸x保证金(%)=基础货币的保证金要求*杠杆保证金公式：交易头寸/杠杆=基础货币的保证金要求*

      *基础货币是一个外汇货币对中第一个报价货币（第一个货币/第二个货币：美元/日元中的美元，欧元/美元中的欧元，等等）。

      例1

      在1,000欧元/美元交易中，保证金要求为0.50%，即杠杆为200:1，则计算如下：

      百分比保证金要求：1,000 x 0.005 = 5.00欧元*

      杠杆保证金要求：1,000 / 200 =5.00欧元*

      *5.00欧元是欧元数量，因为保证金以基础货币计算。在这个例子中，欧元是欧元/美元货币对中的基础货币（欧元=基础货币，美元=计价货币）。如果投资者的账户以非欧元货币计价，系统将自动将保证金要求按当前市场汇率转换成账户货币。

      例2

      1,000美元/日元交易中，保证金要求为0.50%，即杠杆为200:1，计算如下：

      百分比保证金要求：1,000 x 0.005 = 5.00美金*

      杠杆保证金要求：1,000 / 200 =5.00美金*

      *5.00美金是美元数量，因为保证金以基础货币计算。在这个例子中，美元是美元/日元货币对中的基础货币（美元=基础货币，日元=计价货币）。如果投资者的账户以非美元货币计价，系统将自动将点差费用按当前市场汇率转换成账户货币。

      例3

      1,000英镑/加元交易中，保证金要求为0.50%，即杠杆为200:1，计算如下：

      百分比保证金要求：1,000 x 0.0025 = 2.50英镑*

      杠杆保证金要求：1,000 / 400 = 2.50英镑*

      *2.50英镑是英镑数量，因为保证金以基础货币计算。在这个例子中，英镑是英镑/加元货币对中的基础货币（英镑=基础货币，加元=计价货币）。如果投资者的账户以非英镑货币计价，系统将自动将点差费用按当前市场汇率转换成账户货币。

      所有外汇固定交易品种的保证金要求都可以在AVATRADE交易条款中找到，请看上述表格。

      *-0.03美金是美元数量，因为美元是美元/日元货币对中的基础货币（美元=基础货币，日元=计价货币）。如果投资者的账户以非美元货币计价，系统将自动将隔夜利息按当前市场汇率转换成账户货币。

      例3

      1,000英镑/加元交易中，溢价买（或卖）率为-1.00%，并且是1天的金额，则计算如下：

      (1,000 x -0.01 x 1)/360 = -10/360 = -0.02778 = -0.03英镑*（四舍五入）

  • 外汇浮动(只限MT4)

    • 外汇浮动点差计算

      除非另有说明，外汇浮动（仅限MT4）交易条款显示浮动交易品种的最低和标准买卖价差（点子）。标准点差来自上一季度交易时段(格林威治时间07.00-18.00)内各个点差的中值。

      点差成本公式：点差x交易头寸=收取的计价货币*点差费用

      *计价货币是一个外汇货币对中的第二个报价货币（第一个货币/第二个货币：美元/日元中的日元，欧元/美元中的美元，等等）。

      例1

      在1,000欧元/美元交易中，点差为3个点子(0.0003)，则计算如下：

      0.0003 X 1,000 = 0.30美金*

      *0.30美金是美元数量，因为点子以计价货币计算。在这个例子中，美元是欧元/美元货币对中的计价货币（欧元=基础货币，美元=计价货币）。如果投资者的账户以非美元货币计价，系统将自动将点差费用按当前市场汇率转换成账户货币。

      例2

      0.04 X 1,000 = ¥40.00*

      1,000美元/日元交易中，点差为4个点子(0.04)，计算如下：

      0.04 X 1,000 = 40.00日元*

      *40.00日元是日元数量，因为点子以计价货币计算。在这个例子中，日元是美元/日元货币对中的计价货币（美元=基础货币，日元=计价货币）。如果投资者的账户以非日元货币计价，系统将自动将点差费用按当前市场汇率转换成账户货币。

      例3

      0.0012 X 1,000 = C$1.20*

      1,000英镑/加元交易中，点差为12个点子(0.0012)，计算如下：

      (0.0012) X 1,000 = 1.2加元*

    • 外汇浮动隔夜利息计算

      外汇浮动（仅限MT4）交易条款显示，以360天为基础，在我们的日结算日后继续持有未平仓头寸，需要支付/可以得到的隔夜利息。这些利率显示在“溢价买”和“溢价卖”栏。日结算日指格林威治标准时间22:00，夏令时则调整为格林威治标准时间21:00。

      投资者可以用公布的溢价，按以下公式计算自己的每日溢价数额：交易数量x溢价或利率x交易天数=支付/收取的溢价*360天

      *支付/收取的溢价将以基础货币计算；基础货币是一个外汇货币对中第一个报价货币（第一个货币/第二个货币：美元/日元中的美元，欧元/美元中的欧元，等等）。

      例1

      在1,000欧元/美元交易中，溢价买（或卖）率为-1.00%，并且是1天的金额，则计算如下：

      (1,000 x -0.01 x 1)/360 = -10/360 = -0.02778 = -0.03欧元*（四舍五入）

      *-0.03欧元是欧元数量，因为欧元是欧元/美元货币对中的基础货币（欧元=基础货币，美元=计价货币）。如果投资者的账户以非欧元货币计价，系统将自动将隔夜利息按当时的市场汇率转换成账户货币。

      例2

      1,000美元/日元交易中，溢价买（或卖）率为-1.00%，并且是1天的金额，则计算如下：

      (1,000 x -0.01 x 1)/360 = -10/360 = -0.02778 = -0.03美元* （四舍五入）

      *-0.03美元是美元数量，因为美元是美元/日元货币对中的基础货币（美元=基础货币，日元=计价货币）。如果投资者的账户以非美元货币计价，系统将自动将点隔夜利息按当时的市场汇率转换成账户货币。

浮动点差 – 正常市场环境下，交易品种的点差是小幅度浮动的，根据多家银行报价呈现出买价与卖价的最优价格，最小点差为市场可能出现的最低点差，典型点差为经过统计之后的平均点差水平或中间值。

隔夜利息卖出/买入（每日） – 隔夜利息按每种产品每日百分比借记/贷记。
交易时间 – 指定产品的交易时间。
交割月份 – 期货、股指和债券类交易品种涉及到期货主力合约到期交割问题，交割的月份是相对固定的，具体日期时间请查询CFD品种交割展期提醒。
交易所 – 除外汇以外的交易品种，均属于差价合约产品，对应的差价合约标的物价格来自于全球不同知名交易所的期货、股指、股票、ETF和债券等品种。

MT4名称 – 对应交易品种在MT4软件中显示的交易品种名称代码
期权执行价与时间增量 – 根据期权执行价的高低以及期权到期日的不同来确定每个品种的交易成本。
固定到期日 – 期权到期期限的选择是固定的，通常最短为1天，最长达1年。
